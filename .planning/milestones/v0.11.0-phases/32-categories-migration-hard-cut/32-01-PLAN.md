<objective>
Migrate the categories contract from RootConfig (arrconf.yml) into IntentConfig (intent.yml), per D-32-01 and D-32-05. This is the indivisible contract-migration plan: it cannot be done half-way because removing `RootConfig.categories` immediately breaks every generator call site and the diff command at runtime. All signature changes, schema regen, and call-site rewiring land atomically so the Python triade stays green at plan end.

Purpose: Make `intent.yml` the single hand-edited owner of `categories[]` (CATMIG-01). After this plan, `categories[]` can no longer be declared in `arrconf.yml` (RootConfig has no such field), and the pure generators are decoupled from both config types — they take a plain `list[MediaCategory]`.

Output: Updated pydantic schemas, generator signatures, wired apply/diff call sites, regenerated JSON schemas, and the Phase 32 chart image co-bump (0.22.0->0.23.0) — landed here in wave 1 because this is the commit that touches `tools/arrconf/**` (CLAUDE.md same-commit co-bump rule). No file emission yet (that is Plan 02). No data lift of the production categories yet (also Plan 02).
</objective>
