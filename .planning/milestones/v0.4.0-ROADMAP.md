## Overview

Three-part milestone: (a) **finish the v0.3.0 Categories pivot** by ripping the v0.2.0 transition layer (Phase 12), (b) **ship automated content discovery** via SuggestArr (Phases 13–14), and (c) **deliver a local web UI** for editing `arrconf.yml` from any device on the LAN (Phase 15).

After this milestone, the arr-stack operator workflow is:
1. Open the browser at `http://<lan-ip>:8765/`, edit the typed form (categories table + 6 app sections), click "Enregistrer".
2. The UI writes `arrconf.yml` via ruyaml round-trip preserving comments.
3. `git diff` + `git push` from the terminal.
4. ArgoCD picks up; arrconf reconciler propagates to Sonarr/Radarr/Prowlarr/qBittorrent/Seerr/Jellyfin.
5. SuggestArr scans Jellyfin watch history hourly + auto-submits Seerr requests with anime-aware routing.

**Definition of Done achieved:**
- v0.2.0 transition layer fully retired — `merge_with_manual()` removed, 11 flat YAML blocks deleted, pydantic `extra="forbid"` enforces shape (Phase 12 SC#5 dispositive: zero config drift cross-version).
- SuggestArr deployed as 11th `bjw-s/app-template@5.0.0` alias, routing config operator-pasted post-deploy via web UI per RESEARCH-guided architecture (Phase 14 SC#1-5, operator UAT confirmed anime suggestion → `/media/anime` Sonarr rootFolder).
- Local config UI shipped as sibling Python package `tools/arrconf-ui/` — schema-driven form (D-13), no hand-coded per-field UI, French localization, dark theme, LAN-accessible (Phase 15 SC#1-7 + UAT 10/10 scenarios PASSED).

## Lessons learned

1. **frontend-design skill mid-execution is high-value**: the array-of-objects bug + dark theme + French i18n were discovered after the initial Plan 15-B "complete" — applying the skill as a follow-up pass produced a markedly better operator experience without re-planning.

2. **Schema-driven form pattern (D-13) earns its complexity**: walking JSON Schema once + dispatching widgets means adding a new pydantic field to `config.py` shows up automatically in the UI. Zero per-field hand-coding maintenance cost.

3. **Plan-checker BLOCKER prevents architecture drift**: the iter-1 Plan 15-B mis-interpreted RESEARCH and invented a ConfigMap mechanism SuggestArr ignores. Plan-checker caught this in dimension `scope_reduction / research_resolution` before any code shipped. RESEARCH line 488 was the dispositive evidence.

4. **Single-tenant homelab justifies "no auth"**: D-04 amendment to bind 0.0.0.0 + no auth scheme was the right call given the operator's trust model (same LAN, same trust as Sonarr/Radarr already LAN-exposed). Would NOT generalize to multi-tenant or work-LAN contexts.

5. **Mid-UAT scope additions are OK if the contract holds**: 3 unplanned iterations in Phase 15 (LAN bind, design pass, full FR) — each was scoped, each commit was atomic, each iteration left the UAT scenario suite in a clean re-runnable state. Process held up under operator feedback pressure.
