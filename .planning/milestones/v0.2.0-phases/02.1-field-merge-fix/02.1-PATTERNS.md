**Action:** extend existing
**Anchor lines for the change:** add new function between `diff_models` (ends L54) and `reconcile` (starts L57), so insert at L55. The new helper is a sibling of `diff_models` — same module, same logger, same scope.

**Existing `_READ_ONLY_FIELDS` constant (L19-25)** — keep unchanged per D-32 (rule is value-based, not name-based):
```python

    "id",
    "implementationName",
    "infoLink",
    "message",
    "presets",
}
```

**Existing `Action` enum (L28-36)** — UPDATE is the only branch the fix touches:
```python
class Action(Enum):
    """Reconciliation outcomes (D-04 / D-09 / D-11)."""
    ADD = "add"
    UPDATE = "update"
    DELETE = "delete"
    NO_OP = "no-op"
    PRUNE_SKIP = "prune-skip"
    PRUNE_PROTECTED = "prune-protected"
```

**Existing `diff_models()` (L50-54)** — model the new helper on its style (same `model_dump(exclude_none=True, exclude=_READ_ONLY_FIELDS)` discipline):
```python
def diff_models(a: BaseModel, b: BaseModel) -> list[str]:
    """Return sorted field names that differ (excluding D-21 read-only fields)."""
    a_dump = a.model_dump(exclude_none=True, exclude=_READ_ONLY_FIELDS)
    b_dump = b.model_dump(exclude_none=True, exclude=_READ_ONLY_FIELDS)
    return sorted({k for k in (set(a_dump) | set(b_dump)) if a_dump.get(k) != b_dump.get(k)})
```

**Pattern shape for the new helper** (per D-31/D-32/D-33 + Specifics §):
- Signature: `def merge_fields_for_put(current: BaseModel, desired: BaseModel) -> dict[str, Any]:`
- Take `current.model_dump(exclude_none=True)` and `desired.model_dump(exclude_none=True, exclude=_READ_ONLY_FIELDS)` (same exclusion as `diff_models`).
- For `desired_dump["fields"]` (list of `{name, value, type, ...}` dicts — see `FieldKV` model below), walk each entry by `name`. If `value` in YAML is `''` or `None`, replace with the corresponding `current` field's value. Otherwise keep YAML value.
- `tags` must NOT be merged from current — desired's `tags` already includes the auto-managed `managed_tag_id` (sonarr.py L64-71); the fix is **opposite** of "use cluster's tags" for that field. Per CONTEXT.md §code_context "Pitfall — `tags` field path needs careful handling".
- Return the merged dict ready for `client.put(path, id=..., json=body)`.
- `log.info("merge_field_preserved", name=field_name)` per preserved entry (mirror `log.info("plan_action", ...)` style at L82, L87, L104).

**Reusable shape** (must remain generic — D-33 says Phase 3's Radarr/Prowlarr inherit it, so do NOT hardcode the literal string `"fields"` if a cleaner generic emerges; planner's call):
```python
log = structlog.get_logger()  # already at L17

def merge_fields_for_put[T: BaseModel](current: T, desired: T) -> dict[str, Any]:
    """Merge cluster's stored field values into desired body for PUT (D-31/D-32).

    For each entry in desired.fields[], if value is '' or None,
    take the corresponding cluster value (matched by `name`).
    Otherwise keep desired's value.

    `tags` is intentionally NOT merged — desired's tags list legitimately
    overrides cluster's (the reconciler appends managed_tag_id).
    """
    cur_dump = current.model_dump(exclude_none=True)
    des_dump = desired.model_dump(exclude_none=True, exclude=_READ_ONLY_FIELDS)
    cur_by_name = {f["name"]: f for f in cur_dump.get("fields", [])}
    merged_fields = []
    for des_f in des_dump.get("fields", []):
        v = des_f.get("value")
        if v == "" or v is None:
            cur_f = cur_by_name.get(des_f["name"])
            if cur_f is not None and cur_f.get("value") not in ("", None):
                merged = dict(des_f)
                merged["value"] = cur_f["value"]
                log.info("merge_field_preserved", name=des_f["name"])
                merged_fields.append(merged)
                continue
        merged_fields.append(des_f)
    des_dump["fields"] = merged_fields
    return des_dump
```

**Pitfalls:**
- `_READ_ONLY_FIELDS` set MUST stay frozen — D-32 explicitly rejects field-name-based rules.
- `exclude_none=True` in `model_dump` is asymmetric with empty strings (CONTEXT.md §code_context "Established Patterns") — this asymmetry IS the bug. The new helper compensates by checking `v == "" or v is None` explicitly.
- Test the helper in isolation BEFORE wiring it into `_execute` (split commit if planner wants).

### 3. `tools/arrconf/arrconf/dump.py` — drop REDACTED fields (D-36)

**Action:** extend existing
**Anchor lines:** L50-52 (the `items_dumped` line).

**Existing dump emit (L50-52):**
```python

dcs = [DownloadClient.model_validate(x) for x in raw_dcs]
items_dumped = [dc.model_dump(exclude_none=True) for dc in dcs]
```

**Pattern shape after fix** (D-36: drop fields whose Sonarr-side value is `'***REDACTED***'`):
```python
REDACTED_VALUE = "***REDACTED***"  # Sonarr's privacy stand-in for password/apiKey/userName fields

def _drop_redacted_fields(dc_dump: dict[str, Any]) -> dict[str, Any]:
    """Drop fields[] entries whose value is REDACTED so dump→apply round-trip stays clean (D-36)."""
    if "fields" not in dc_dump:
        return dc_dump
    dc_dump["fields"] = [f for f in dc_dump["fields"] if f.get("value") != REDACTED_VALUE]
    return dc_dump

# in dump_sonarr:
items_dumped = [_drop_redacted_fields(dc.model_dump(exclude_none=True)) for dc in dcs]
```

**Pitfalls:**
- `'***REDACTED***'` is the literal string Sonarr returns for `privacy: password` fields — VERIFIED in `tests/fixtures/sonarr/downloadclient.json` line 65 (the `password` field).
- The dump filter and the merge helper (file #1) are **two ends of the same contract**: dump omits → apply preserves. Both must ship together so round-trip stays at 0 actions (per ROADMAP success criterion #4 of Phase 2.1).
- After this filter, the YAML committed in `examples/baseline-sonarr.yml` (Phase 1 artifact) and `my-kluster/charts/arrconf/files/arrconf.yml` (Phase 2 artifact) will have FEWER `fields[]` entries — that's intentional. Re-run `arrconf dump` to refresh `examples/baseline-sonarr.yml` if the planner wants the committed baseline to reflect the new emit shape (optional clean-up; not blocking).

**Optional generalization:** If the helper grows to handle `apiKey`, `token`, etc. as Sonarr's other privacy categories, extend `REDACTED_VALUE` to a set: `REDACTED_VALUES = {"***REDACTED***"}` (Sonarr currently uses only one sentinel; verified via fixture).
