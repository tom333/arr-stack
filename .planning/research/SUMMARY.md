# Project Research Summary

**Project:** arr-stack v0.9.0
**Domain:** homelab media-stack automation — configarr-in-UI (REQ-config-ui-multi-config) + Jellyfin intro skipper (REQ-jellyfin-skip-intro)
**Researched:** 2026-05-27
**Confidence:** HIGH (stack + architecture verified against live files and upstream sources; MEDIUM on Kodi feasibility and plugin install endpoint)

## Executive Summary

v0.9.0 adds two independent features to the arr-stack: a configarr.yml editor tab inside arrconf-ui, and declarative Intro Skipper plugin management in the Jellyfin arrconf reconciler. Both are well-bounded. Feature B (Jellyfin skip-intro) is mechanically simpler — it extends the existing `_reconcile_plugins()` path with a two-phase install model and adds a plugin repository entry to `arrconf.yml` — and is the recommended first-to-ship. Feature A (configarr-in-UI) is architecturally broader, decomposing naturally into three sub-phases: backend schema + endpoints, frontend form + tab selector, and specialized pickers (TRaSH CF + Recyclarr templates).

The single highest-risk item across both features is `!env`/`!secret` tag preservation on the configarr.yml write path. The existing arrconf-ui IO layer already handles this correctly for arrconf.yml; the configarr path must reuse that exact same `YAML(typ='rt')` helper — any deviation (safe_load, a second YAML instance, a shallow dict merge on `sonarr.main`) silently drops the tags and lands raw API keys in git. This must be implemented and tested before any other UI form work proceeds.

The second scoped decision is whether to build the TRaSH/Recyclarr picker in v0.9.0 or defer it. The live `configarr.yml` uses 6 hand-rolled French-language custom formats and has no `include:` section. A Recyclarr template picker would introduce a merge-order hazard that could silently overwrite those custom CFs. The recommendation is to ship the TRaSH CF name-to-trash_id lookup picker (safe: read-only catalog, no structural change to configarr.yml), and to explicitly exclude Recyclarr template `include:` insertion from v0.9.0.

## Key Findings

### Recommended Stack

No new Python or npm dependencies are needed for either feature. The existing stack (Python 3.13, httpx, pydantic v2, ruyaml 0.91, FastAPI, structlog, pytest/respx, Svelte 5, Vite, TypeScript) is sufficient end-to-end. The configarr backend pydantic model (`ConfigarrRootConfig`) lives in `tools/arrconf-ui/` only — never in `tools/arrconf/` (ADR-5 boundary). The TRaSH and Recyclarr catalogs are build-time TypeScript constants generated by a fetch script; the bundled assets add a few KB, not a new library.

**Core additions (no new pip/npm deps):**

- `ConfigarrRootConfig` pydantic model in `tools/arrconf-ui/arrconf_ui/configarr_config.py` — schema source for the configarr form
- `schemas/configarr-schema.json` — hand-authored JSON Schema, mirrors `schemas/arrconf-schema.json` pattern
- `tools/scripts/fetch-trash-metadata.sh` — dev-time fetch script; pulls TRaSH-Guides + Recyclarr catalogs at pinned commit SHA; output committed as static assets
- `POST /Packages/Installed` Jellyfin API call inside `_reconcile_plugins()` — gated on `install: true` in YAML and plugin absence in `GET /Plugins`
- Intro Skipper: GUID `c83d86bb-a1e0-4c35-a113-e2101cf4ee6b`, version `1.10.11.19`, manifest `https://intro-skipper.org/manifest.json` (Jellyfin 10.11.8 compatible, verified live 2026-05-27)

**Version compatibility confirmed:** configarr 1.28.0 deployed; ruyaml 0.91 `!env`/`!secret` TaggedScalar round-trip verified live; Jellyfin 10.11.8 matches Intro Skipper targetAbi; Media Segments API available since 10.10.

### Expected Features

**Must have (table stakes):**
- Read and display `configarr.yml` in a structured form (quality profiles, custom formats, scores per profile)
- Save `configarr.yml` with `!env`/`!secret` tags preserved — blocking correctness requirement
- Pydantic validation before write (configarr pydantic model in arrconf-ui)
- Diff preview before save (reuse existing `POST /api/diff` pattern)
- Intro Skipper plugin repository + install declared in `arrconf.yml` and reconciled by arrconf
- `EnableChapterImageExtraction: true` set via arrconf library config (seek-bar thumbnails, works in all clients including Kodi)

**Should have (differentiators):**
- TRaSH CF picker: searchable by name, shows default score alongside operator score — eliminates manual hex `trash_id` copying. Safe to build (read-only catalog, no merge hazard).
- Kodi salon skip-intro: `service.jellyskip` documented as operator runbook (outside arrconf scope); binary spike with accept/reject criterion

**Defer to v2+:**
- Recyclarr template `include:` insertion via the picker — changes configarr.yml structure, merge-order hazard for the hand-rolled CF setup (see Cross-Document Tensions)
- Clone/rename quality profiles — homelab has a stable 3-profile set
- `media_naming` field editing — show read-only; rarely touched
- Server-side forced auto-skip — opt-in only; documented as "extreme cases only", overrides all clients

### Architecture Approach

Feature A extends arrconf-ui with 4 parallel configarr endpoints plus 2 read-only TRaSH/Recyclarr metadata endpoints. The frontend gains a config selector tab; the existing `FieldInput.svelte` schema-driven dispatcher is reused unchanged; two new specialized Svelte components handle the TRaSH picker and Recyclarr reference dropdown. The io.py write path is unchanged — the existing `YAML(typ='rt')` + `write_yaml_atomic` already handles tagged scalars; the constraint is to never introduce a second YAML instance or shallow dict replacement of a tagged subtree (`sonarr.main`, `radarr.main`). Feature B is a pure arrconf reconciler extension: two new YAML fields (`install: true`, `id`, `version` on plugin entries) and a new `POST /Packages/Installed` call gated on plugin absence.

### Critical Pitfalls

1. **`!env`/`!secret` tag drop on configarr.yml write-back (A1, CRITICAL)** — Use the exact same `YAML(typ='rt')` instance for load and dump; never replace `target['sonarr']` with a plain Python dict. Add a test loading the real `charts/arr-stack/files/configarr.yml`, writing to a temp file, asserting tags present in output bytes — before any other UI write work.

2. **UI exposes resolved secrets in diff endpoint (A2, SECURITY)** — Treat `!env SONARR_API_KEY` as an opaque string at all times in arrconf-ui. Never resolve env vars in any endpoint response. Test that `/api/configarr/diff` returns the literal string `!env SONARR_API_KEY`, not a resolved value.

3. **Plugin install requires Jellyfin restart — reconciler cannot verify post-restart in same run (B3)** — Model two distinct states: "repository registered + package queued" (idempotence via repositories list) and "plugin active" (idempotence via `GET /Plugins`). Operator restarts once manually; do not automate from the CronJob.

4. **Kodi/JellyCon has no native Media Segments support — issue #953 open (B1, FEASIBILITY GATE)** — Kodi skip-intro requires `service.jellyskip` (third-party Kodi addon, manual LibreELEC install, outside arrconf scope). Treat as best-effort; run a spike with a binary accept/reject criterion. Do not gate the milestone on Kodi.

5. **configarr has no published JSON Schema — hand-authored schema drifts from reality (A4)** — Model only the fields the UI actively edits; do not build a deep full-structure clone. Add a CI gate: `configarr --dry-run` against saved YAML must exit 0.

6. **TRaSH metadata staleness and Recyclarr/configarr template incompatibility (A5, A6)** — Bundle TRaSH catalog as a pinned snapshot at a known commit SHA. Fetch Recyclarr templates from a configarr-compatible commit (v7.4.0 baseline), not HEAD. Display `trash_id` + `name` together; warn on unknown IDs.

7. **Duplicate plugin repository entries on re-runs (B6)** — GET-diff check on repositories by URL (exact string match) before POST. Same pattern as all other arrconf resource types.

## Cross-Document Tensions and Resolutions

### Tension 1: TRaSH/Recyclarr Picker Scope (OPEN SCOPE DECISION)

STACK.md and ARCHITECTURE.md treat both the TRaSH CF picker and Recyclarr template picker as in-scope for v0.9.0 (build-time baked catalogs, no new deps). FEATURES.md classifies Recyclarr template `include:` insertion as a P3 anti-feature, noting the live `configarr.yml` has no `include:` section and merging templates into a hand-rolled file with 6 custom French CFs creates unpredictable merge-order hazards.

**Recommended default:** Split the scope. Build the **TRaSH CF name-to-trash_id picker** (safe: read-only catalog lookup, operator gets a human-readable name → hex `trash_id` is inserted into `custom_formats[].trash_ids`). Defer **Recyclarr template `include:` insertion** past v0.9.0 — the `RecyclarrTemplatePicker.svelte` component shows template names as informational reference only, without offering to insert `include:` blocks. This should be confirmed as a requirements-phase decision.

### Tension 2: `!env`/`!secret` Round-Trip — Phase-Zero Guard

All four documents agree. STACK.md confirms ruyaml 0.91 handles `TaggedScalar` correctly when `typ='rt'` is used consistently. FEATURES.md identifies the shallow top-key merge as the specific failure mode for `sonarr.main`. PITFALLS.md rates recovery as HIGH cost (secret in git history, requires rebase + key rotation). ARCHITECTURE.md confirms io.py needs no modification — the constraint is reuse, not rework.

**Resolution:** Phase 25, task 0. The `!env` preservation test ships before any other configarr write-path code.

### Tension 3: Kodi/JellyCon Media Segments Feasibility

JellyCon issue #953 is open; native support does not exist. `service.jellyskip` is the only viable workaround. Server-side forced auto-skip overrides all clients and is not a default.

**Resolution:** Kodi is best-effort. Milestone acceptance: skip button on web + Android app. Kodi result documented after the Phase 24 spike (accept = `service.jellyskip` verified on LibreELEC 10.11.8; reject = unsupported, documented, not a blocker).

### Tension 4: Plugin Install Requires Operator Restart

All docs agree: no automation from CronJob. Reconciler logs warning after queuing install; operator runs `kubectl rollout restart deployment/jellyfin -n selfhost` once. Second reconciler run transitions to activation path.

## Implications for Roadmap

v0.9.0 decomposes into 4 phases. Feature B first (smaller, tighter test discipline, validates Jellyfin plugin API before UI work). Feature A splits along its dependency chain.

### Phase 24: Feature B — Jellyfin Intro Skipper (arrconf reconciler)

**Rationale:** Smaller scope, no frontend, fits arrconf test discipline. Validates Jellyfin plugin API interaction (including the MEDIUM-confidence `POST /Packages/Installed` endpoint) before the larger UI work begins.

**Delivers:**
- `_reconcile_plugins()` extended with two-phase install model (`install: true` flag; only calls `POST /Packages/Installed` when plugin absent in `GET /Plugins`)
- `arrconf.yml` updated: Intro Skipper repo (`https://intro-skipper.org/manifest.json`) + plugin entry (GUID + version pinned)
- `EnableChapterImageExtraction: true` in Jellyfin library config
- arrconf image co-bump (CLAUDE.md pin pattern)
- Operator runbook: first run → `kubectl rollout restart` → second run (activation) → trigger "Media Segment Scan" manually
- Kodi spike: `service.jellyskip` on LibreELEC — binary accept/reject criterion documented

**Avoids:** B3 (two-phase state), B6 (GET-diff idempotence), B2 (correct manifest URL + ffmpeg version check)

**Research flag:** Confirm `POST /Packages/Installed` exact parameter format against live Jellyfin 10.11.8 as the first implementation step. Confirm `EnableChapterImageExtraction` LibraryOptions POST path format.

### Phase 25: Feature A — configarr-in-UI Backend (schema + endpoints)

**Rationale:** The `!env` guard and pydantic model are prerequisites for all frontend work. Backend must be tested in isolation first.

**Delivers:**
- `configarr_config.py`: `ConfigarrRootConfig` pydantic model (covers quality_profiles, custom_formats, assign_scores_to, quality_definition; excludes specifications[] raw editing and multi-instance keys)
- `schemas/configarr-schema.json`: hand-authored; `api_key` fields marked `readOnly: true`
- 4 new endpoints: `GET/PUT /api/configarr/config`, `POST /api/configarr/diff`, `GET /api/configarr/schema`
- `locator.py` extensions (3 new path helpers)
- `!env`/`!secret` preservation test (load real configarr.yml → write to temp → assert tags intact) — task 0
- 32+ backend tests for new endpoints

**Avoids:** A1 + A2 (`!env` guard tested first), A3 (no *arr API URLs in arrconf-ui), A4 (model scoped to edited fields only)

**Research flag:** Standard patterns. No research phase needed. Recommend running `configarr --dry-run` against the hand-authored schema output as a sanity check (confirm `--dry-run` flag exists in configarr 1.28.0 first).

### Phase 26: Feature A — configarr-in-UI Frontend (form + config selector)

**Rationale:** Depends on Phase 25 endpoints. Reuses existing Svelte components without modification; work is wiring + tab selector.

**Delivers:**
- `App.svelte`: config selector tab (`arrconf | configarr`)
- `api.ts` + `types.ts`: configarr API functions and TypeScript types (`ConfigarrPayload`, etc.)
- Configarr form: quality profiles, custom formats, scores per profile; `media_naming` read-only
- Diff preview reusing existing diff component
- Frontend smoke tests (Vitest)

**Avoids:** A3 (no *arr API calls on frontend), Recyclarr include insertion (deferred per Tension 1)

**Research flag:** Standard patterns (Svelte 5 schema-driven form). No research phase needed.

### Phase 27: Feature A — TRaSH CF Picker + Recyclarr Template Reference

**Rationale:** Specialized components depend on Phase 26 form and the bundled metadata catalog.

**Delivers:**
- `tools/scripts/fetch-trash-metadata.sh`: fetches TRaSH-Guides + Recyclarr catalogs at pinned commit SHA; committed as `tools/arrconf-ui/web/src/assets/trash-metadata/`
- 2 read-only metadata endpoints (`GET /api/trash/custom-formats?app=sonarr|radarr`, `GET /api/trash/quality-profiles?app=sonarr|radarr`)
- `TrashPicker.svelte`: searchable multi-select for `custom_formats[].trash_ids`; shows `name` + TRaSH default score; warns on unknown IDs
- `RecyclarrTemplatePicker.svelte`: informational reference dropdown (does NOT insert `include:` blocks — per Tension 1 resolution)
- Backend tests for metadata endpoints

**Avoids:** A5 (pinned snapshot, not live fetch), A6 (Recyclarr catalog sourced from configarr-compatible ≤ v7.4.0 commit), anti-pattern 1 (no runtime GitHub calls from FastAPI)

**Research flag:** Identify the exact `recyclarr/config-templates` commit SHA corresponding to configarr's v7.4.0 fork point before writing the fetch script.

### Phase Ordering Rationale

- Feature B first: single-codebase (arrconf Python), validates the live Jellyfin plugin API before building the UI. The two-run validation cycle (install → restart → activate) is better discovered early.
- Phases 25 → 26 → 27: strict dependency chain — backend schema before frontend form before specialized pickers.
- Phase 24 and Phase 25 backend work are fully independent and can run in parallel if staffing allows.
