C'est tout. **NE PAS ajouter** :
- `customManagers` regex sur `values.yaml` → DEFERRED en Phase 4 (le fichier n'existe pas encore — Renovate fail validation si on cible un fichier inexistant) ;
- `packageRules` (rien à régler tant qu'aucune dépendance n'est trackée) ;
- `extends: [":semanticCommits"]` ou autres presets opinionated (laisser Phase 4 décider).

Justification de `config:recommended` (vs `config:base` déprécié) : confirmé current dans la doc officielle Renovate (`docs.renovatebot.com/config-presets/`), warn de validation si on utilise `config:base` depuis Renovate v33+ (~2023). Cf 00-RESEARCH.md "State of the Art".

**Validation post-écriture** :

1. Le JSON doit être strictement valide :
   ```bash
   jq empty renovate.json
   ```
   Doit exit 0 (sinon JSON cassé).

2. La clé `extends` doit contenir `config:recommended` :
   ```bash
   jq -e '.extends | index("config:recommended")' renovate.json
   ```
   Doit exit 0 et imprimer un index >= 0.

3. Validation officielle Renovate via npx (pas d'install globale, juste téléchargement temporaire) :
   ```bash
   npx --yes --package=renovate -- renovate-config-validator renovate.json
   ```
   Doit imprimer `Validating renovate.json` et `Config validated successfully` (ou équivalent), exit 0.

Si `npx` n'a pas de réseau (cas dégradé), tomber back sur `jq` validation seul + référence au CI Phase 1+ qui re-validera. Pas un blocker hard si le validator officiel n'est pas joignable.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Filesystem → Git | Les fichiers créés ici seront committés. Tout secret oublié devient public si le repo est rendu public. |
| Renovate config → Renovate runtime (GitHub) | `renovate.json` est lu par le bot Renovate sur GitHub. Erreur de syntaxe = bot ne tourne pas (échec silencieux). |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-00-01 | Information Disclosure | `.gitignore` (absence de pattern correct) | mitigate | `.gitignore` ignore `.env*` et `*.cookies` / `*.cookie-jar` (D-17). Vérification automatique : `grep -q '*.cookies' .gitignore && grep -q '.env' .gitignore`. Couvre T1 (API keys env) et T2 (cookies qBittorrent) listés dans le contexte de planning. |
| T-00-02 | Information Disclosure | `.gitignore` ignorant `snapshots/` par erreur | mitigate | Test négatif : `! grep -qE '^snapshots/?$' .gitignore`. Commentaire explicatif "NE PAS ignorer snapshots/" dans le fichier pour empêcher un futur dev de le rajouter sans réfléchir (D-04). |
| T-00-03 | Tampering | `renovate.json` mal formé (Renovate ne tourne plus) | mitigate | Validation triple : `jq empty` (syntaxe JSON), `jq -e '.extends \| index(...)'` (contenu attendu), `renovate-config-validator` (sémantique Renovate). |
| T-00-04 | Information Disclosure | README pointant vers fichier secret par accident | accept | README ne référence que des fichiers publics (spec.md, CLAUDE.md, tools/snapshot/README.md, .planning/) — pas de risque concret. |

Ce plan ne touche pas le cluster ni les APIs externes : T1-T4 du contexte de planning (leak via process listing, leak via cookies, leak via dump API, élévation accidentelle write) sont **non applicables ici** (T1/T2/T3/T4 sont mitigés par les Plans 02 et 03).
