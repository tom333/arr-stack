**`load_intent` pattern** (lines 183-201) — unchanged, already wraps `ConfigError`:
```python
def load_intent(path: Path) -> IntentConfig:
    if not path.exists():
        raise ConfigError(f"Intent file not found: {path}")
    yaml = YAML(typ="safe")
    try:
        with path.open("r", encoding="utf-8") as f:
            raw = yaml.load(f) or {}
    except Exception as e:
        raise ConfigError(f"YAML parse error in {path}: {e}") from e
    try:
        cfg = IntentConfig.model_validate(raw)
    except ValidationError as e:
        raise ConfigError(f"Intent validation error in {path}: {e}") from e
    return cfg
```

### `tools/arrconf/arrconf/generators/categories.py` (pure-generator, transform)

**Analog:** self — signatures change from `cfg: RootConfig` to `cfg: IntentConfig` (or accept `list[Category]` directly).

**Context:** After CATMIG, `categories[]` lives in `intent.yml` / `IntentConfig`. The 5 generator functions currently take `cfg: RootConfig` and access `cfg.categories`. They must accept the new source. Two options:
1. Change parameter to `list[MediaCategory]` (most KISS — callers pass `intent_cfg.categories`).
