## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| operator-authored intent.yml → arrconf generate | Untrusted YAML (profile_definitions + configarr skeleton) parsed into IntentConfig |
| arrconf generate → committed configarr.yml | Generated file committed to git, read by configarr at apply time |
| configarr.yml → live Sonarr/Radarr APIs | configarr (NOT arrconf) applies; arrconf must never cross this boundary (ADR-5) |

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-33-05 | Information Disclosure | api_key in committed configarr.yml | mitigate | Generated file keeps api_key as bare `!env SONARR_API_KEY`/`!env RADARR_API_KEY` tag — never an expanded secret. acceptance_criteria greps assert bare tag present + zero quoted/expanded form. BLOCK-WORTHY: secret leakage into committed GENERATED file. |
| T-33-06 | Tampering | intent.yml → configarr.yml drift (stale committed file) | mitigate | CI generate-idempotence (`generate --check` + drift flag) gates configarr.yml; a PR editing intent without regenerating fails. CFGARR-04. |
| T-33-07 | Repudiation | hand-edits to GENERATED configarr.yml going unnoticed | mitigate | GENERATED header + CI drift gate make manual edits fail CI; hard cut removes the double-source ambiguity. |
| T-33-08 | Denial of Service | image tag / GHCR mismatch (ImagePullBackOff) | accept | Co-bump aligns values.yaml with the image; no push/tag from planner avoids the auto-tag race (MEMORY: accumulated-bumps escape hatch). Operator-controlled release. |
