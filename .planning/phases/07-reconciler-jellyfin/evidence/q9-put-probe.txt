## Section 2 — POST /System/Configuration is full REPLACE (NOT merge) — Pitfall 1

**Endpoint:** POST /System/Configuration (PUT not supported — OpenAPI evidence below)

**Body shape:** Full ServerConfiguration object (56 properties, all optional per OpenAPI required: []).

**Round-trip behavior:**
- POST full GET body verbatim → HTTP 204, all fields preserved → idempotent OK
- POST partial body {"ServerName":"jellyfin"} only → HTTP 204, **other fields RESET to defaults**:
  - UICulture: "fr" → "en-US"  (live evidence)
  - MetadataCountryCode: "FR" → "US"
  - PreferredMetadataLanguage: "fr" → "en"
  - PluginRepositories: [{"Name":"Jellyfin Stable",...}] → []  (wiped!)

**Reconciler implication (CRITICAL):** The reconciler MUST follow pattern GET → mutate allowlist 7 fields → POST entire body.
The merge is done ARRCONF-SIDE, not Jellyfin-side. Any field omitted from body becomes its C# default value.

**OpenAPI evidence (ServerConfiguration schema):**
- required: [] (no fields required — Jellyfin fills C# defaults if omitted)
- 56 properties total — including MetadataOptions, TrickplayOptions, CodecsUsed, CastReceiverApplications, SortRemoveWords, etc.

## Section 3 — POST /Library/VirtualFolders/Paths NOT idempotent — Pitfall 2 (CRITICAL)

**Endpoint:** POST /Library/VirtualFolders/Paths?refreshLibrary=false
**Body (MediaPathDto):** {"Name":"<library-name>","Path":"<path>","PathInfo":{"Path":"<path>"}}
**OpenAPI:** required: ['Name']; Path + PathInfo nullable.

| Scenario | HTTP | Result |
|----------|------|--------|
| Path NOT in library | 204 | Path added to PathInfos |
| Path ALREADY in library | 204 | **Path DUPLICATED in PathInfos** (NOT idempotent — verified live: PathInfos: ['/media/series', '/media/series']) |
| Library Name does not exist | 404 | Error processing request. |

**Reconciler implication (CRITICAL):** Before POST, the reconciler MUST:
1. GET /Library/VirtualFolders for the library
2. Extract LibraryOptions.PathInfos[].Path as a set
3. Skip POST if path is already in the set (idempotence shim arrconf-side)

**Cleanup evidence (probe restored Séries library):** PathInfos ['/media/series'] confirmed final state.

## Section 4 — DELETE /Library/VirtualFolders/Paths removes ALL matching entries — Pitfall 3

**Endpoint:** DELETE /Library/VirtualFolders/Paths?name=<lib>&path=<path>&refreshLibrary=false

| Scenario | HTTP | Result |
|----------|------|--------|
| Path exists exactly once | 204 | Removed |
| Path exists DUPLICATED (e.g. after non-idempotent POST) | 204 | **ALL matching entries deleted** (verified: from ['/media/series','/media/series'] → []) |
| Path does NOT exist | 204 (silent) | No effect |
| Library Name does not exist | 404 | Error processing request. |

**Implication:** If Pitfall 2 created a duplicate, DELETE wipes both entries. Reconciler must treat PathInfos as a conceptual set, never a multilist.

**Secondary observation:** Locations (read-only display field) shows stale duplicates after path manipulation but PathInfos (source of truth used for indexing) is authoritative. Library refresh OR Jellyfin pod restart syncs the two (non-blocking for reconciliation).

## Section 5 — POST /Users/{userId}/Policy — POST not PUT — Pitfall 4

**Endpoint:** POST /Users/{userId}/Policy (PUT returns HTTP 405 Method Not Allowed — verified live)

**Body (UserPolicy):** Full 44-property object, required: ['AuthenticationProviderId', 'PasswordResetProviderId'].

**Round-trip:**
- POST current Policy verbatim → HTTP 204 idempotent OK
- PUT current Policy → HTTP 405 Method Not Allowed

**Reconciler implication:** MUST use client.post(...) not client.put(...).
JellyfinClient does NOT inherit from _ArrV3Client so it does NOT inherit the forceSave wrap — clean.

**Critical D-06-OPENAPI-01 carry-forward (REQUIRED OpenAPI fields — Pitfall 6):**
- AuthenticationProviderId: "Jellyfin.Server.Implementations.Users.DefaultAuthenticationProvider" — REQUIRED by OpenAPI.
  If excluded by pydantic, POST returns HTTP 400. MUST preserve from GET cluster (pattern Seerr apiKey).
- PasswordResetProviderId: "Jellyfin.Server.Implementations.Users.DefaultPasswordResetProvider" — REQUIRED by OpenAPI.
  MUST preserve from GET cluster.

**Replace vs merge semantics (UserPolicy):** Hypothesis (strong, based on Pitfall 1 — System/Configuration full REPLACE):
POST /Users/{id}/Policy is likely also full REPLACE. Reconciler must follow same GET → mutate allowlist → POST entire body pattern.
If probe confirmation differs, adjust pattern accordingly.

## Section 6 — POST /Plugins/{pluginId}/{version}/Enable — Version REQUIRED in path — Pitfall 5

**Initial guess (CONTEXT.md):** POST /Plugins/{id}/Enable → **WRONG**, returns HTTP 405.

**Actual endpoint (OpenAPI verified):** POST /Plugins/{pluginId}/{version}/Enable

| Probe | HTTP |
|-------|------|
| POST /Plugins/{id}/Enable (no version in path) | 405 Method Not Allowed |
| POST /Plugins/{pluginId}/{version}/Enable (TMDb b8715... + 10.11.8.0) on already-Active plugin | 204 idempotent OK |

**Reconciler implication (CRITICAL):** Reconciler MUST:
1. GET /Plugins to obtain Status, Id, Version per plugin Name
2. Resolve Name → (Id, Version) tuple
3. If Status != "Active" → POST /Plugins/{Id}/{Version}/Enable

**Complementary endpoint:** POST /Plugins/{pluginId}/{version}/Disable (404 on unknown ID).

**PluginStatus enum values (OpenAPI 10.11.8):** ['Active', 'Restart', 'Deleted', 'Superseded', 'Superceded', 'Malfunctioned', 'NotSupported', 'Disabled']

Activation logic: If Status is NOT in {'Active', 'Restart'} → attempt Enable.
'Restart' = Active but requires Jellyfin restart (transient, idempotent → no-op).
