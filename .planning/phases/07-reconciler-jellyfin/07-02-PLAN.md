<objective>
Wave 1 of Phase 7 — pydantic models + config integration + JSON Schema regen. This plan is the **type-layer foundation** for Plan 07-04 (reconciler implementation) and Plan 07-05 (chart YAML wiring).

The 4 pydantic resource modules under `resources/jellyfin/` codify the 4 reconcile scopes (D-07-LIB-01, D-07-USERS-01, D-07-CONFIG-01, D-07-PLUGINS-01) at the type level. `config.py` gets the section/instance hierarchy + `RootConfig.jellyfin` field. `settings.py` gets the `JELLYFIN_API_KEY` env var binding. JSON Schema is regenerated for VS Code autocomplete (REQ-yaml-autocomplete).

**Critical anti-patterns to honour (RESEARCH-verified pitfalls):**
- **Pitfall 6 (D-07-CREDS-01 / RESEARCH §296-309)**: `UserPolicy` OpenAPI 10.11.8 requires `AuthenticationProviderId` AND `PasswordResetProviderId`. These fields MUST be `Field(exclude=True)` in the pydantic model (operator does not set them in YAML — Pitfall 6 symmetry with Seerr apiKey D-06-CREDS-01). The reconciler (Plan 07-04) re-injects them from cluster GET before POST.
- **D-07-LIB-02**: Library model scope is strictly `name + collection_type + paths`. LibraryOptions fields (TypeOptions, EnableRealtimeMonitor, MetadataLanguage per-library, etc.) are out-of-scope — they MUST NOT appear in the pydantic model.
- **D-07-CONFIG-01**: Server config allowlist = EXACTLY 7 fields. All others (TrickplayOptions, MetadataOptions, CodecsUsed, etc.) stay operator-managed. The pydantic model exposes 7 fields + `extra="allow"` for forward-compat (Phase 6 D-06-OPENAPI-01 carry-forward — the cluster GET will contain ~49 other fields that flow through the reconciler-side merge in Plan 07-04, not via this model).
- **D-07-INSTANCE-01 + ADR-7**: Single-instance pattern. `RootConfig.jellyfin: dict[str, JellyfinInstance]` always has exactly one key, `"main"`.
- **REQ-prune-opt-in**: `JellyfinLibrariesSection.prune` and `JellyfinUsersSection.prune` default to `False`. D-07-USERS-01 hardcodes prune=false (emilie protection) — the field exists for symmetry with other phases but is NEVER True in Phase 7 YAML.

Plan 07-03 (test fixtures) runs in parallel with this plan (no file overlap). Plan 07-04 depends on BOTH 07-02 and 07-03.

Output: 9 files modified or created. ~400 LOC of pydantic + config + ~80 LOC of new tests.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| YAML config (operator-authored) → load_config → RootConfig.jellyfin | First parse boundary. Pydantic `extra="forbid"` on Section/Instance models catches typos at load time (D-07-INSTANCE-01 safety). |
| RootConfig.jellyfin → schema-gen → JSON Schema → VS Code autocomplete | The schema is the operator-facing contract. Any field added to the pydantic model with a non-exclude=True declaration becomes a YAML autocomplete suggestion. |
| pydantic model_dump() output → reconciler (Plan 07-04) PUT body | The `Field(exclude=True)` on AuthenticationProviderId + PasswordResetProviderId is the FIRST line of defense — the model_dump cannot leak operator-authored values into the PUT body (only reconciler-side re-injection from cluster GET is allowed). |
