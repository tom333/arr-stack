    ```yaml
    cross-seed:
      global:
        nameOverride: cross-seed
        fullnameOverride: cross-seed
      serviceAccount:
        cross-seed: {}
      controllers:
        main:
          type: deployment   # D-03: daemon mode, long-running, port 2468
          replicas: 1
          initContainers:
            config-init:

                repository: ghcr.io/cross-seed/cross-seed
                tag: "6.13.7"

              command:
                - node
                - -e
                - |
                  const fs = require('fs');
                  let c = fs.readFileSync('/config-cm/config.js', 'utf8');
                  c = c.replace(/\$\{PROWLARR_API_KEY\}/g, process.env.PROWLARR_API_KEY || '');
                  c = c.replace(/\$\{QBT_USER\}/g, process.env.QBT_USER || '');
                  c = c.replace(/\$\{QBT_PASS\}/g, process.env.QBT_PASS || '');
                  fs.writeFileSync('/config-resolved/config.js', c);
              envFrom:
                - secretRef:
                    name: arrconf-env
          containers:
            main:
              image:
                # renovate: image=ghcr.io/cross-seed/cross-seed
                repository: ghcr.io/cross-seed/cross-seed
                tag: "6.13.7"
                pullPolicy: IfNotPresent
              args:
                - daemon
              env:
                TZ: "Europe/Paris"
              envFrom:
                - secretRef:
                    name: arrconf-env
              probes:
                liveness:
                  enabled: true
                  custom: true
                  spec:
                    tcpSocket:
                      port: 2468
                    initialDelaySeconds: 30
                    periodSeconds: 30
                readiness:
                  enabled: true
                  custom: true
                  spec:
                    tcpSocket:
                      port: 2468
                    initialDelaySeconds: 30
                    periodSeconds: 30
              resources:
                limits:
                  cpu: 250m
                  memory: 256Mi
                requests:
                  cpu: 50m
                  memory: 128Mi
      service:
        main:
          controller: main
          ports:
            http:
              port: 2468
      # NO ingress block — cluster-internal only (mirror suggestarr D-14); access via port-forward.
      persistence:
        # Read-only ConfigMap with the ${...}-tokenised config.js. Mounted ONLY into the
        # initContainer (it is the substitution SOURCE; the main container never reads it).
        config-cm:
          type: configMap
          name: cross-seed-config
          advancedMounts:
            main:
              config-init:
                - path: /config-cm/config.js
                  subPath: config.js
                  readOnly: true
        # Writable scratch emptyDir. The initContainer WRITES the resolved file to
        # /config-resolved/config.js; the main container SURFACES that same file at
        # /config/config.js via subPath so it does NOT shadow the PVC at /config (Pitfall 1).
        config-resolved:
          type: emptyDir
          advancedMounts:
            main:
              config-init:
                - path: /config-resolved
              main:
                - path: /config/config.js
                  subPath: config.js
        # PVC holds cross-seed state (config.db, torrent_cache/). Mounts at /config in the
        # main container only; the subPath file mount above sits inside it without shadowing it.
        config:
          existingClaim: cross-seed-config
          advancedMounts:
            main:
              main:
                - path: /config
        torrents:
          type: hostPath
          hostPath: /media/data/torrents
          hostPathType: DirectoryOrCreate
          advancedMounts:
            main:
              main:
                - path: /data
    ```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| ConfigMap (committed, read-only) → initContainer → emptyDir (runtime) | Secret values cross ONLY at runtime inside the pod, never into the committed ConfigMap or git |
| arrconf-env SealedSecret → pod env | Real PROWLARR_API_KEY / QBT_USER / QBT_PASS enter the pod via envFrom; sourced from the cluster SealedSecret |
| cross-seed daemon API (port 2468) | Cluster-internal only; no ingress |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-30-04 | Information Disclosure | cross-seed-configmap.yaml + values.yaml in git | mitigate | ConfigMap renders config.js with `${...}` tokens ONLY (sourced from Plan 01's committed artifact); values.yaml references the `arrconf-env` SealedSecret by name via `envFrom`, embeds NO secret literal. Acceptance grep asserts no resolved secret in rendered manifest's ConfigMap |
| T-30-05 | Information Disclosure | resolved config.js with real secrets | mitigate | Resolved file lives in an `emptyDir` (in-memory, ephemeral); never persisted to the PVC or committed. initContainer re-runs each pod start. PVC at /config holds only cross-seed DB/cache, never config.js source-of-truth |
| T-30-06 | Tampering | ConfigMap read-only contract | mitigate | configMap volume mounted `readOnly: true` (into the initContainer only); the initContainer writes to a separate `config-resolved` emptyDir, never back to the ConfigMap |
| T-30-07 | Denial of Service | initContainer fails → empty secret values | accept | `process.env.X || ''` yields empty string if a key is missing; cross-seed fails auth (visible in logs) rather than crashlooping silently. Operator pre-req: confirm QBT_USER/QBT_PASS/PROWLARR_API_KEY exist in arrconf-env (Plan 03 runbook) |
| T-30-08 | Elevation of Privilege | hostPath /media/data/torrents | accept | Identical hostPath as the existing qBittorrent alias (established trust); required for hardlink integrity (D-05). DirectoryOrCreate, no new privilege surface |
</threat_model>
