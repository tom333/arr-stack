# Phase 15 — UI Design Contract

> Visual and interaction contract for the local config UI frontend.
> Generated by gsd-ui-researcher. Single-tenant homelab config editor —
> calibrated for lean visual ambition: one operator, episodic use, laptop browser only.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none — vanilla Svelte scoped styles |
| Preset | not applicable |
| Component library | none — hand-rolled form components per pydantic section |
| Icon library | none — Unicode symbols for action buttons (↑ ↓ ✕); SVG inline only if a coupling badge icon is used |
| Font | system-ui stack: `system-ui, -apple-system, BlinkMacSystemFont, "Segoe UI", sans-serif` |

Source: CONTEXT.md Claude's Discretion — "CSS framework: Tailwind, Pico CSS, vanilla CSS, or Svelte scoped styles. The UI has ~50 fields organized in ~7 sections — vanilla scoped styles likely suffice."

No Tailwind. No CSS reset library. No shadcn (React-specific — not applicable). Svelte `<style>` scoped blocks are the sole styling mechanism.

## Spacing Scale

Declared values (multiples of 4):

| Token | Value | CSS var | Usage |
|-------|-------|---------|-------|
| xs | 4px | `--space-xs` | Icon gaps, badge padding, inline chip spacing |
| sm | 8px | `--space-sm` | Input inner padding (horizontal), row gap within a field group |
| md | 16px | `--space-md` | Default element spacing, label-to-input gap, table cell padding |
| lg | 24px | `--space-lg` | Section padding (collapsible panel interior), header bar vertical padding |
| xl | 32px | `--space-xl` | Layout gap between major sections (categories table ↔ app sections) |
| 2xl | 48px | `--space-2xl` | Top-of-page margin above header bar |
| 3xl | 64px | `--space-3xl` | Reserved — not used in Phase 15 surfaces |

Exceptions:

- **Touch targets** (↑ ↓ ✕ action buttons in the categories table): minimum 32px × 32px hit area regardless of font size. Achieved via `min-width: 32px; min-height: 32px; padding: var(--space-xs)`.
- **Diff preview panel**: 8px internal padding on the semantic summary list items (matches `--space-sm`).
- **Toast notification**: 12px top/bottom × 16px left/right padding (non-standard mid-point — acceptable for a transient overlay).

## Typography

System font stack only. No web font loading. Two font sizes, two weights.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 14px | 400 (regular) | 1.5 | All form inputs, labels, table cell text, section prose, error messages, toast text |
| Label / small | 12px | 400 (regular) | 1.4 | Field helper text, badge text ("SuggestArr"), diff count sub-labels, collapsible section sub-title |
| Section heading | 16px | 600 (semibold) | 1.2 | Collapsible section title (`<summary>` text), Categories table heading |
| App heading | 20px | 600 (semibold) | 1.2 | Page title ("arrconf editor") in the header bar |

Notes:
- Exactly 4 sizes, 2 weights — no deviations.
- No italic, no underline except for links (there are no navigation links in this UI).
- `font-size: 14px` is the `<body>` default — all relative sizing (`em`, `rem`) is derived from this base.
- Input elements inherit body font-size explicitly (`font: inherit` on `<input>`, `<select>`, `<textarea>`).

## Surface Inventory and Interaction States

### 1. Header bar

**Layout:** Full-width sticky bar at top. Left: app title "arrconf editor" (20px semibold) + current file path (`charts/arr-stack/files/arrconf.yml`) in 12px muted text. Right: diff-count chip + Save button.

**Save button states:**
- `disabled` (no pending changes): `opacity: 0.4`, `cursor: not-allowed`. Label: "Save".
- `enabled` (changes pending): accent fill, white label. Label: "Save".
- `loading` (PUT in flight): accent fill, spinner icon replacing label. No second click possible.

**Diff-count chip:**
- Hidden when no pending changes.
- Shown as `"N unsaved change(s)"` in 12px muted text, left of Save button, when any form field differs from the loaded config.
- No accent color — muted/neutral chip. Draws attention via proximity to Save, not by color.

**Source:** CONTEXT D-12 (launch UX), ROADMAP Phase 15 SC#5 (Save shows indicator).

### 2. Categories editor table

**Layout:** Full-width table. Columns: Name, Kind (dropdown), Profile (dropdown), Display (text), Base path (text), Actions (↑ ↓ ✕).

**Row interaction:**
- `↑` button: disabled on first row (`opacity: 0.4`, `cursor: not-allowed`). Active: 32×32px hit target, `--color-muted` fill, moves row up on click.
- `↓` button: disabled on last row (same). Active: same, moves row down.
- `✕` button: default state `--color-muted`, hover state `--color-destructive`. Single click → inline confirmation replaces action buttons: `[Confirm delete] [Cancel]` text buttons, no modal. Source: CONTEXT D-08.
- Row hover: `background: #f1f5f9` (slate-100 tint).
- Alternate row: even rows use `--color-surface` (`#f8f9fa`), odd rows use `--color-panel` (`#ffffff`).

**Inline add-row form (bottom of table):**
- Always visible below the last data row (not a modal).
- Fields: Name (text), Kind (select: `series`/`movies`), Profile (select: `general`/`anime`/`family`), Display (text), Base path (text).
- Action buttons: "Add" (accent fill) + "Reset" (text button, no fill).
- Source: CONTEXT D-08 — "inline new-row form (NOT a modal)".

**Kind dropdown options:** `series`, `movies` (Literal from `MediaCategory.kind`).
**Profile dropdown options:** `general`, `anime`, `family` (derived from category shapes in `arrconf.yml`).

### 3. Per-app collapsible sections

**Layout:** 6 sections (sonarr, radarr, prowlarr, qbittorrent, seerr, jellyfin), each rendered as a native `<details>` element. Section title in 16px semibold as `<summary>`.

**Open/closed state:**
- Collapsed (default on load): `<summary>` shows section name + field count chip ("12 fields").
- Expanded: section interior visible, section padding `--space-lg` (24px).

**Field rendering by pydantic type:**
- `str` → `<input type="text">`
- `int` → `<input type="number" step="1">`
- `bool` → `<input type="checkbox">` with inline label
- `list[int]` → comma-separated text input (e.g., `tags: [2, 3]` displayed as `2, 3`)
- `Literal["fullSync", "addOnly", "disabled"]` → `<select>` with the literal options
- `bool` `enable` / `prune` fields → checkbox; `prune` checkbox includes a tooltip warning: "Caution: enabling prune deletes unmanaged resources on next reconcile."

**Nested sub-sections** (e.g., `sonarr.main.content_routing.rules[]`):
- Rendered as a sub-table within the section (same columns: tag, keywords).
- ↑ ↓ ✕ row actions apply (same interaction contract as categories table).
- "Add rule" button at the bottom of the sub-table.

**Instance key** (`main` in `sonarr.main`): displayed as a read-only chip label above the section fields. The operator does not rename instances via the UI.

**source:** CONTEXT D-01 (Svelte 5 vanilla), REQ-local-config-ui-frontend.

### 4. Validation error display

**Per-field inline error:**
- Input border changes to `--color-error-border` (`#ef4444`).
- Below the input: 12px text in `--color-destructive`, prefix "Error: ".
- Error text is the pydantic `msg` from the 422 response's `errors[i]` array, matched by `loc` path to the field.
- Fields with errors have a `background: var(--color-error-bg)` tint on their `<input>`.

**Top-of-page error summary banner:**
- Appears below the header bar when PUT /api/config returns 422.
- Copy: "N validation error(s) — fix the highlighted fields before saving."
- Background: `var(--color-error-bg)`, border-left: 4px solid `var(--color-destructive)`.
- Dismissible via ✕ button on the right.
- Auto-clears on the next successful Save.

**Source:** CONTEXT D-06 (validation on Save only), REQ-local-config-ui-frontend SC#5.

### 5. SuggestArr coupling badge

**Appearance:** Small inline badge next to field label, right-aligned or immediately to the right of the field name text. Text: "↗ SuggestArr". Background: `var(--color-badge-bg)` (`#eff6ff`). Text color: `var(--color-badge-fg)` (`#1d4ed8`). Border-radius: 4px. Font: 12px regular. Padding: 2px 6px.

**Tooltip (hover or focus):** Native `title` attribute — "Linked to SuggestArr's SEER_ANIME_PROFILE_CONFIG (Phase 14 D-05/D-06/D-07). Changing this value requires re-pasting routing config in SuggestArr's web UI per `evidence/derived-routing-values.md`."

**Fields receiving the badge (7 total per CONTEXT D-09):**
1. `seerr.main.sonarr_service.activeAnimeProfileId`
2. `seerr.main.sonarr_service.activeProfileId`
3. `seerr.main.sonarr_service.activeAnimeDirectory`
4. `seerr.main.sonarr_service.activeDirectory`
5. `seerr.main.radarr_service.activeProfileId`
6. `seerr.main.radarr_service.activeDirectory`
7. `categories[name="films-zoe"].base_path` (the anime-movie rootFolder derivation per CONTEXT D-09)

**Behavior:** INFORMATIONAL ONLY. The field remains fully editable. No lock, no disable, no confirm step. Source: CONTEXT D-09.

### 6. Diff preview modal/panel

**Trigger:** Clicking the "Save" button (when enabled) opens the diff panel BEFORE writing the file.

**Presentation:** Inline panel below the header bar (NOT a modal overlay — avoids focus-trap complexity for a single-user tool). Panel slides in / appears with a 150ms fade-in transition.

**Content layout:**
- Panel heading: "Pending changes — review before saving" (16px semibold).
- Per-section structured list. Each section with changes shows:
  - Section name (bold, 14px)
  - Sub-items: `+ 2 categories added: series-thomas, films-zoe` / `~ 1 modified: sonarr.main.content_routing.rules[0].keywords` / `- 1 removed: legacy-cat`
  - Sections with no changes are hidden (not "0 changes" entries).
- If no changes detected: "No changes detected" (muted text) + disabled Save button (this state should not be reachable normally since Save is disabled without changes, but defensive case).

**Actions:**
- "Confirm & Save" button (accent fill). Triggers PUT /api/config.
- "Cancel" text button (no fill). Closes the panel, returns to editing.

**Source:** CONTEXT D-07 (semantic summary format), ROADMAP Phase 15 SC#4.

### 7. Save toast notification

**Trigger:** Successful PUT /api/config response (HTTP 200).

**Appearance:** Fixed-position bottom-right corner. Background `--color-panel` (`#ffffff`). Border: 1px solid `--color-border`. Border-left: 4px solid accent (`#3b82f6`). Border-radius: 6px. Shadow: `0 2px 8px rgba(0,0,0,0.12)`. Padding: 12px 16px.

**Copy:** "Saved — run `git diff` to review, then push."

**Behavior:**
- Auto-dismiss after 4 seconds.
- Dismiss immediately on click.
- Only one toast shown at a time (subsequent saves replace the existing toast timer).

**Typography:** 14px regular. The `git diff` fragment is rendered in `<code>` tags (monospace, `--color-muted` background).

**Source:** CONTEXT D-05 (no git automation, toast copy), REQUIREMENTS REQ-local-config-ui-frontend SC#6.

### 8. Empty / loading states

**Initial page load (GET /api/config in flight):**
- Full-page centered spinner (CSS animation, no library).
- Copy below spinner: "Loading arrconf.yml…" (14px muted).
- Duration: typically < 50ms for a local file read; visible only on slow I/O.

**GET /api/config error (file missing or parse failure):**
- Error banner full width. Copy: "Could not load arrconf.yml — [error message from backend]. Check the file path and try again."
- No form rendered. No Save button.

**No pending changes (Save button disabled):**
- Save button visually disabled (`opacity: 0.4`). No tooltip needed — the diff-count chip being absent is sufficient affordance.

**Empty categories list (edge case):**
- Categories table body: single row spanning all columns with muted text "No categories defined. Use the form below to add one."
- "Add row" inline form still visible and active.

## Keyboard Accessibility

Source: CONTEXT D-08 — "keyboard-accessible" explicitly required for the ↑ ↓ actions.

| Interaction | Keyboard contract |
|---|---|
| Tab navigation | All form inputs, buttons, selects, and collapsible `<summary>` elements are in the natural tab order. No `tabindex` overrides. |
| Category row ↑ / ↓ | Buttons receive focus; Enter/Space activate them. |
| Category row delete | ✕ button receives focus; Enter/Space shows inline confirm. Confirm/Cancel buttons are next in tab order. |
| Diff panel | Panel appears in DOM above the form; first focusable element is "Confirm & Save". Escape key dismisses panel (returns focus to Save button). |
| Collapsible sections | Native `<details>`/`<summary>` — browser-native keyboard support (Enter/Space to toggle). |
| Toast | Not focusable — auto-dismiss only. No interactive elements inside toast. |
| Error banner | Dismiss ✕ button receives focus; Enter/Space dismisses. |

**State management:** Svelte 5 runes API (`$state`, `$derived`, `$effect`). No external store library. Root state lives in `App.svelte`:
- `configState` — the in-memory edited config object (mirror of loaded JSON).
- `savedConfig` — the last-saved config (for diff computation and "hasChanges" derivation).
- `validationErrors` — array of pydantic error objects from the last 422 response.
- `saveStatus` — `"idle" | "saving" | "saved" | "error"`.

## API Integration (frontend ↔ backend)

| Endpoint | Frontend event | Expected response |
|---|---|---|
| `GET /api/config` | Page load | 200 JSON — full config object |
| `PUT /api/config` | "Confirm & Save" click | 200 JSON `{diff: {...}}` or 422 JSON `{detail: [{loc, msg, type}]}` |
| `GET /api/diff` (optional, if backend exposes it) | "Save" button hover or click — compute diff preview | 200 JSON diff structure per CONTEXT D-07 |

If `GET /api/diff` is not implemented in 15-A, the frontend computes the semantic diff client-side by comparing `configState` vs `savedConfig` using the same structural comparison logic. The backend diff is the authoritative one (used post-save for the confirmed diff summary); the frontend diff is the preview.

**Error handling:**
- 422: parse `detail` array, map each `loc` path to the corresponding form field, set red border + error text.
- Network error / 500: show error banner: "Save failed — server error. Check the terminal running arrconf-ui."
