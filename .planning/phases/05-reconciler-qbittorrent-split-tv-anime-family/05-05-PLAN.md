<objective>
Extend `reconcile_sonarr` to ship the ADR-7 split for Sonarr: 3 new tags (`tv`, `anime`, `family`), 3 root folders (existing `/media/series` kept + 2 new), 3 download clients with `tags: [<label>]` tag-routing, the NEW resource type `remote_path_mappings` (3 entries for the split + keep existing `/data/complete/`), and the NEW retroactive content tagging via `PUT /api/v3/series/editor` (D-05-MIG-01).

CRITICAL ordering invariant (D-05-ORDER-01): tags MUST be reconciled BEFORE download_clients (because download_client.tags carries tag IDs that don't exist yet otherwise), AND series_tags MUST run AFTER download_clients (otherwise a series tagged `tv` could route to an unrouted client and land in the wrong save_path). This plan codes the ordering explicitly inside `reconcile_sonarr` and ships a regression test that fails if the order is ever permuted.

Per D-05-PATHS-01, the chart-side YAML (Plan 07) keeps `/media/series` as the existing root folder — only adds `/media/anime` and `/media/family`. The Sonarr reconciler in this plan is generic; the specific values come from `charts/arr-stack/files/arrconf.yml`.

Purpose: Land the Sonarr half of REQ-app-coverage (split tv/anime/family) and the retroactive content tagging. Plan 06 mirrors this for Radarr. SC#3 unit-level signal is here; cluster-side dispositive is Plan 08.

Output: 1 MODIFIED reconciler (sonarr.py — 3 new sub-functions + label-resolver + ordering glue) + 1 EXTENDED + 2 NEW test modules + 1 EXTENDED scope-violation test.
</objective>
