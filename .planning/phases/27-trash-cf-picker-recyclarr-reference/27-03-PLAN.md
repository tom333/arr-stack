<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| baked catalog data → DOM render | trash_description contains upstream HTML (`<br>`) |
| picker → in-memory config → PUT | new entries cross into the persisted configarr.yml |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-27-09 | Tampering | unknown trash_id silently dropped on save | mitigate | CF picker never rewrites entries it does not recognize; unknown ids preserved verbatim, removed only by explicit ✕ (SC#4 / D-03); grep gate ensures no auto-strip |
| T-27-10 | XSS / injection | `trash_description` HTML rendered via `{@html}` | mitigate | Prefer text-render (`<br>`→newline) over `{@html}`; data source is the committed baked catalog (no operator/network input), so worst case is static TRaSH text — low surface |
| T-27-11 | Tampering | QP picker overwriting/reordering hand-rolled profiles | mitigate | Append-only via spread; grep gate forbids `existingProfiles[idx]=`/`.splice`/`.sort`; collision blocks same-name insert (D-10) |
| T-27-12 | Spoofing (ADR-5) | picker constructing a *arr URL | mitigate | Pickers only call `/api/trash/*` (relative); no *arr host string introduced (grep gate in Plan 04) |
</threat_model>
