## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| operator browser → arrconf-ui backend | LAN-trusted single operator; JSON payloads cross here (intent edits) |
| backend → local filesystem | intent.yml + generated files written to charts/arr-stack/files/ |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-34-01 | Tampering | PUT /api/intent file write path | mitigate | Write only to `intent_yml_path()` / `arrconf_yml_path()` / `configarr_yml_path()` — fixed locator-derived paths, no operator-supplied path component. No path traversal surface. |
| T-34-02 | Tampering | intent.yml deserialization | mitigate | Read via `load_intent` (ruyaml safe load inside arrconf) + validate with `IntentConfig` (`extra="forbid"`). No `yaml.load`/unsafe loader. |
| T-34-03 | Elevation | regenerate path dialing a *arr API | mitigate | Save/diff call only pure `generate_*` functions + file writes; ADR-5 boundary comment on every handler; no httpx/URL construction. base_url echoed verbatim, never called. |
| T-34-04 | Tampering | editing a generated file via removed PUT | mitigate | PUT /api/config + PUT /api/configarr/config removed (D-34-04). Generated files are write-targets only of the regenerate step, never operator-editable. |
| T-34-05 | Spoofing/Repudiation | no auth on local tool | accept | LAN-trusted single-operator design (CONTEXT.md). No auth/multi-user by design. Out of scope. |
</threat_model>
