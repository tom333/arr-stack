## Color

All tokens pre-exist in `app.css`. Two themes (`[data-theme="light"]` / `[data-theme="dark"]`). Executor uses CSS variables — never hardcoded hex.

| Role | Light value | Dark value | CSS variable | Usage |
|------|-------------|------------|--------------|-------|
| Dominant surface (60%) | `#faf9f6` | `#0b0e14` | `--surface` | Page background, read-only inspector backgrounds |
| Panel / secondary (30%) | `#ffffff` | `#11161c` | `--panel` | Cards, DiffPanel, dialogs, form cards |
| Panel alt | `#f4f1ea` | `#161c24` | `--panel-alt` | Alternate rows, CF chips, read-only field backgrounds |
| Accent (10%) | `#0891b2` | `#22d3ee` | `--accent` | Reserved elements only — see below |
| Destructive | `#b91c1c` | `#f87171` | `--destructive` | Destructive action indicators, delete buttons on hover |
| Doc card bg | `#fff8e6` | `#1a1208` | `--doc-bg` | SectionDoc card backgrounds |
| Doc card border | `#d97706` | `#d97706` | `--doc-border` | SectionDoc left rail accent |

**Accent (`--accent`) is reserved for exactly:**
1. The "Enregistrer" (save) button background and active tab indicator — pre-existing pattern
2. The diff chip ("N modification(s) en attente") in HeaderBar
3. Focus rings (`outline: 2px solid var(--accent)` — pre-existing)
4. The "Confirmer et enregistrer" button in the materialization diff panel
5. The "~" operation marker for modified lines inside the unified diff display
6. Active/selected state of the per-profile picker "add" button

**Read-only inspector treatment (legacy arrconf.yml / configarr.yml views):**
- Background: `--surface` (not `--panel`) — visually depressed relative to editable panels
- Fields: `opacity: 0.6`, `pointer-events: none`, `cursor: not-allowed` on inputs
- No accent, no save button — inspection only

**Semantic colors in unified diff:**
- Added lines (`+`): `#10b981` (emerald) — pre-existing in DiffPanel `.op-add`
- Modified lines (`~`): `var(--accent)` — pre-existing in DiffPanel `.op-mod`
- Removed lines (`-`): `var(--destructive)` — pre-existing in DiffPanel `.op-del`
- Line background for `+` lines: `rgba(16, 185, 129, 0.08)` (emerald at 8% opacity)
- Line background for `-` lines: `rgba(var(--destructive), 0.08)` — dark: `rgba(248, 113, 113, 0.08)`

## Information Architecture

### Page Structure

The page has one top-level layout, unchanged from the existing pattern:

```
┌─────────────────────────────────────────────────────────────────┐
│ HeaderBar (sticky, position:fixed top)                          │
│  arrconf/editor   [intent.yml path]   [diff chip] [Save btn]    │
│  [tab: intent.yml | arrconf.yml (inspect) | configarr.yml (inspect)] │
└─────────────────────────────────────────────────────────────────┘
┌─────────────────────────────────────────────────────────────────┐
│ .page (max-width: 960px, centered, --space-lg margin)           │
│  ValidationBanner (if errors)                                   │
│  MaterializationDiffPanel (if open — replaces DiffPanel)        │
│  ─── active tab content ──────────────────────────────────────  │
│  [intent tab:]                                                  │
│    SectionDoc(intent.categories)                                │
│    CategoriesEditor (REUSED AS-IS)                              │
│    SectionDoc(intent.sagas)                                     │
│    SagasEditor (new, schema-driven via AppSection)              │
│    SectionDoc(intent.apps)                                      │
│    AppsPassthroughEditor (raw textarea or schema-driven)        │
│    SectionDoc(intent.tools)                                     │
│    ToolsEditor (schema-driven via AppSection)                   │
│    SectionDoc(intent.profile_definitions)                       │
│    ProfileDefinitionsEditor (special-cased — see below)         │
│    SectionDoc(intent.configarr)                                 │
│    ConfigarrRawEditor (opaque textarea — see below)             │
│  [arrconf.yml inspect tab:]                                     │
│    ReadOnlyInspector(arrconf.yml)                               │
│  [configarr.yml inspect tab:]                                   │
│    ReadOnlyInspector(configarr.yml)                             │
└─────────────────────────────────────────────────────────────────┘
```

### Tab Navigation

HeaderBar tabs become 3 entries (replacing the current 2):

| Tab label | `font-family` | Behaviour |
|-----------|---------------|-----------|
| `intent.yml` | IBM Plex Mono 12px | Default active tab; all editing happens here |
| `arrconf.yml` | IBM Plex Mono 12px | Read-only inspector; no save button shown |
| `configarr.yml` | IBM Plex Mono 12px | Read-only inspector; no save button shown |

Active tab: `--accent-soft` background, `--accent` text, `--accent` border (pre-existing `.tab-active` class — extend to 3 tabs).

Unsaved-change guard applies when leaving `intent.yml` tab if `diffCount > 0` (reuse existing `confirmSwitchOpen` dialog pattern verbatim).

The "Enregistrer" save button is **only visible when the `intent.yml` tab is active**. On inspect tabs it is hidden (not just disabled).

The HeaderBar `filePath` display shows:
- `intent.yml` tab: `charts/arr-stack/files/intent.yml`
- `arrconf.yml` tab: `charts/arr-stack/files/arrconf.yml` + `[généré — lecture seule]` badge
- `configarr.yml` tab: `charts/arr-stack/files/configarr.yml` + `[généré — lecture seule]` badge

The `[généré — lecture seule]` badge: inline `<span>` after the `<code class="filepath">`, styled as the existing `.diff-chip` shape (pill, `--panel-alt` bg, `--ink-muted` text, `IBM Plex Mono` 12px) — no accent color (not actionable).

## Intent Form Sections

### Section ordering in intent.yml tab

1. `categories` — CategoriesEditor (reused as-is, no changes)
2. `sagas` — list editor via schema-driven AppSection
3. `apps` — schema-driven AppSection (pass-through dict, display as-is)
4. `tools` — schema-driven AppSection (cross_seed + qbit_manage sub-blocks)
5. `profile_definitions` — **special-cased ProfileDefinitionsEditor** (see below)
6. `configarr` — **special-cased ConfigarrRawEditor** (see below)

Each section is preceded by a `SectionDoc` card (reused pattern). New `SECTION_DOCS` entries for `intent.categories`, `intent.sagas`, `intent.apps`, `intent.tools`, `intent.profile_definitions`, `intent.configarr` must be added to `src/i18n/fr.ts`.

### ProfileDefinitionsEditor (special-cased)

Layout: one collapsible card per profile name (MULTi.VF, Anime, Family). Each card is a `<details>` styled to match the existing `SectionDoc` card pattern (same `--panel` background, `--border` border, `border-radius: 4px`).

```
┌─ profile: MULTi.VF ──────────────────────────────── [▶] ──┐
│  body: [raw YAML textarea — IBM Plex Mono 12px]            │
│  custom_formats:                                           │
│    [CF chip list — existing chip style from TrashCFPicker] │
│    TrashCFPicker (mounted here — for this profile only)    │
│    TrashQPPicker (mounted here — for this profile only)    │
└────────────────────────────────────────────────────────────┘
```

- The `body` field is a raw `<textarea>` with `IBM Plex Mono` 12px. The operator edits QP fields (language, upgrade, qualities, etc.) as YAML text. No structured form — the body is `dict[str, Any]` opaque from arrconf's perspective.
- The `custom_formats` list renders as CF chips (identical to existing `TrashCFPicker` chip style: `--panel-alt` background, `--border` border, `IBM Plex Mono` 12px, `✕` delete button).
- Below the chip list: `TrashCFPicker` and `TrashQPPicker` components **mounted per profile** with `app` inferred from profile kind (sonarr-profiles → `sonarr`, radarr-profiles → `radarr`; if ambiguous, show an `app` selector). These are the existing v0.9.0 components reused without modification.
- Score override: next to each CF chip, a compact `<input type="number">` (width 5em, `IBM Plex Mono`) shows the `score` field. Empty = use CF default. Placeholder: `défaut`.
- Add profile button: `+ Ajouter un profil` — same style as the existing `array-add` pattern (`--accent-soft` background, `--accent` border, `IBM Plex Sans` 13px). Prompts for profile name via an inline text input.
- Delete profile button: `✕` small button at card header right, `--ink-muted` color, `--destructive` on hover. No confirmation dialog (not destructive enough for a separate confirmation).

### ConfigarrRawEditor (special-cased)

A single `<textarea>` bound to `intent.configarr` serialized as YAML. The backend round-trips it as `dict[str, Any]` verbatim.

- Font: `IBM Plex Mono` 12px
- Min-height: 200px; auto-grow on content (or fixed 320px with scroll)
- Background: `--code-bg` (matches the existing `code` element background — `#f4f1ea` light, `#161c24` dark)
- Border: `1px solid var(--border)`, `border-radius: 3px`, focus ring `--accent`
- Label above textarea: "Bloc configarr pass-through (YAML brut)" at 13px `IBM Plex Sans` weight 500 `--ink`
- Helper text below: "Ce bloc est émis verbatim dans configarr.yml. arrconf-ui ne le valide pas." at 12px `--ink-faint` italic.

## Materialization Diff Panel

This replaces and extends the existing `DiffPanel` for Phase 34.

The new panel shows **two labelled file diffs** — arrconf.yml and configarr.yml — in unified text diff format (not the existing semantic field-level diff). This is a new component: `MaterializationDiffPanel`.

### Layout

```
┌────────────────────────────────────────────────────────────┐
│ h2: "Matérialisation — vérifier avant d'enregistrer"       │
│                                                            │
│ ─── arrconf.yml ─────────────────────────────────────────  │
│ [unified diff block — scrollable, max-height 300px]        │
│                                                            │
│ ─── configarr.yml ───────────────────────────────────────  │
│ [unified diff block — scrollable, max-height 300px]        │
│                                                            │
│ [Continuer l'édition]          [Confirmer et enregistrer]  │
└────────────────────────────────────────────────────────────┘
```

- Panel background: `--panel`, border `1px solid var(--border)`, `border-radius: 4px`, `padding: var(--space-lg)`, `box-shadow: var(--shadow-md)` — identical to existing DiffPanel.
- File label (e.g. "arrconf.yml"): `code` element at 12px `IBM Plex Mono` `--ink-muted`, preceded by a thin horizontal rule (`border-top: 1px solid var(--border)`), `margin: var(--space-md) 0 var(--space-sm) 0`.
- Unified diff block: `<pre>` element, `overflow-y: auto`, `max-height: 300px`, background `--code-bg`, `border: 1px solid var(--border)`, `border-radius: 3px`, `padding: var(--space-sm) var(--space-md)`, `font-size: 12px`, `IBM Plex Mono`.

### Diff line coloring

| Line prefix | Text color | Background |
|-------------|------------|------------|
| `+` | `#10b981` | `rgba(16, 185, 129, 0.08)` |
| `-` | `var(--destructive)` | dark: `rgba(248, 113, 113, 0.08)` / light: `rgba(185, 28, 28, 0.06)` |
| `@@` hunk header | `var(--accent)` | `var(--accent-soft)` at 30% opacity |
| context lines | `var(--ink-muted)` | transparent |

### "No changes" state

If a generated file is identical to its on-disk version: show a single line "Aucune modification" in `--ink-faint` italic inside the diff block, instead of an empty `<pre>`.

### Actions

Two buttons, right-aligned in a `flex gap: var(--space-sm)` row with a top border `1px solid var(--border)` and `padding-top: var(--space-md)` — identical to existing DiffPanel actions layout.

| Button | Style |
|--------|-------|
| "Continuer l'édition" | Default button style — `--panel` bg, `--border` border, `--ink` text |
| "Confirmer et enregistrer" | `--accent` bg, `--accent-fg` text, `--accent` border, weight 500 |

## Read-Only Inspector

Rendered when the `arrconf.yml` or `configarr.yml` tab is active.

- Shows the **on-disk file content** as a `<pre>` block (raw YAML).
- Styling: `IBM Plex Mono` 12px, background `--code-bg`, `border: 1px solid var(--border)`, `border-radius: 4px`, `padding: var(--space-lg)`, `overflow-y: auto`, no max-height (full content visible, page scrolls).
- Above the block: file label in `code` 12px + the `[généré — lecture seule]` badge (same as HeaderBar treatment).
- No form inputs, no save, no validation — inspection only.
- Load error treatment: same `.load-error` pattern (red left-border card) as existing App.svelte error handling.

## Interaction States

### Intent form field states (reuse existing app.css patterns)

| State | Visual |
|-------|--------|
| Default | `--panel` bg, `--border` border |
| Hover | `--border-strong` border |
| Focus | `--accent` border, `0 0 0 3px var(--accent-soft)` shadow |
| Error / validation | `--error-border` border, `--error-bg` background |
| Read-only (inspect tabs) | `--surface` bg, `opacity: 0.6`, `pointer-events: none` |

### Save button states (unchanged)

| State | Visual |
|-------|--------|
| idle, no changes | disabled, `opacity: 0.4` |
| idle, has changes | `--accent` bg, `--accent-fg` text, enabled |
| saving | disabled, label `Enregistrement…` |
| saved | short `SaveToast` (existing component), button resets to idle |
| error | `ValidationBanner` shown (existing component) |

### Diff chip ("N modification(s) en attente")

Behaviour unchanged from existing: appears in HeaderBar when `diffCount > 0`. Phase 34 changes `diffCount` semantics: it reflects whether the in-memory intent payload differs from the last-saved intent (not per-section JSON compare). Implementation detail left to executor.
