## Deviations

### Deviation 1: D-05.1-BUMP-01 — Renovate App not installed on arr-stack repo

**Discovered:** 2026-05-15 during G-05.1-3 execution (45 min post-merge of PR #9, no Renovate PR appeared; investigation revealed zero `app/renovate` PRs in repo history).

**Cause:** D-05.1-BUMP-01 inferred "arr-stack Renovate" was active because `renovate.json` exists and chart-lint validates it. But the renovate-config-validator step only validates the config syntax — it doesn't run Renovate itself. The Mend Renovate App was never installed on `tom333/arr-stack`.

**Resolution:** Manual values.yaml bump via PR #10. The semantic intent (chart points to freshly published Phase-5 image so Plan 05-08 can run it) is preserved. PR #10's chart-lint run will also create `v0.3.2` (because chart-only changes still trigger the auto-tag chain) — that's harmless: `:0.3.2` will exist on GHCR but unreferenced. Steady-state side-effect of every chart-only PR until Renovate is installed.

**Follow-up (operator-deferred):** Install Mend Renovate App at https://github.com/apps/renovate on `tom333/arr-stack`. After install, the next image bump becomes automatic again.

### Deviation 2 (NONE for A1-ASSUMED): GHCR Probe 1 returned 401, not 200

This is a misread of the planner's expected behavior, not a real deviation. The planner's RESEARCH §SC#4 assumed `401 = exists but auth-gated` was the success path on GHCR's v2 API. That's correct for any tag, real or fake, because the v2 API requires bearer-auth for manifest reads regardless of repository public-status. Probe 1 returning 401 anonymously is plan-acceptable. The supplemental Probe 4 (bearer-auth, returns 200) is the definitive existence check and confirms the chain worked.

The `A1-ASSUMED-REGRESSION` marker path was NOT taken (Probe 4 200 is the success branch; the 404 + marker path was for the empty-string-fallback regression scenario, which did not occur).
