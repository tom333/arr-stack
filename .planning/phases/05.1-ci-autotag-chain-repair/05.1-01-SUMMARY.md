## Deviations from Plan

### Minor — Verification Command Discrepancy (T2)

**[Not a Rule 1-3 deviation — plan verification grep was slightly off]**
- **Found during:** Task 5 validation
- **Issue:** The plan's acceptance criterion for T2 used `grep -B1 "gh api" .github/workflows/chart-lint.yml | grep -q "new_tag != ''"`. With `-B1`, this only looks 1 line before `gh api` (the `run: |` line), which does not contain the `if:` condition (which is 5 lines before `gh api` due to `continue-on-error:`, `env:`, `GH_TOKEN`, `run: |` in between).
- **Fix:** The guard is correctly placed as `if: steps.tag.outputs.new_tag != ''` on the dispatch step's `if:` key (per YAML schema for GitHub Actions steps). The Task 5 evidence file uses `grep -q "if: steps.tag.outputs.new_tag != ''"` directly, which correctly passes.
- **Verdict:** Plan intent fully met. The verification command in the plan was approximate; actual logic is correct and verified.
- **Files modified:** None (verification method adjusted, not implementation)
