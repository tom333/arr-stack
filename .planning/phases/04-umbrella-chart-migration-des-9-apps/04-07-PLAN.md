## Déploiement

### Pré-requis

- Cluster `my-kluster` opérationnel avec ArgoCD installé et `selfhost-project` créé
- Secrets `arrconf-env` + `configarr-env` appliqués dans le namespace `selfhost` (cf [`my-kluster/secrets/`](https://github.com/tom333/my-kluster/tree/main/secrets))
- PVCs existants pour les 8 apps + `configarr-cache` (déjà présents si tu migrais depuis l'état pré-Phase 4)

### Premier déploiement

1. Cloner ce repo (lecture seule) :
   ```bash
   git clone https://github.com/tom333/arr-stack.git
   cd arr-stack
   ```
2. Vérifier le chart localement :
   ```bash
   helm dependency update charts/arr-stack/
   helm lint charts/arr-stack/ -f examples/values-prod.yaml
   helm template arr-stack charts/arr-stack/ -f examples/values-prod.yaml | head -50
   ```
3. Côté `my-kluster` : merger un commit ajoutant [`argocd/argocd-apps/arr-stack-app.yaml`](https://github.com/tom333/my-kluster/blob/main/argocd/argocd-apps/arr-stack-app.yaml). ArgoCD synchronise.
4. **AUCUN `helm install` direct depuis ce repo** — toujours via my-kluster + ArgoCD (cf [`CLAUDE.md`](./CLAUDE.md) "Ce que tu NE dois PAS faire").

### Mise à jour d'image

Renovate ouvre automatiquement une PR de bump dès qu'une image suivie via `# renovate: image=<repo>` change upstream. Workflow :
1. PR Renovate ouverte sur ce repo (`values.yaml` : `tag: 4.0.17` → `4.0.18`)
2. CI `chart-lint.yml` : helm lint + kubeconform + annotation/latest/cronJobConfig guards + regex synthetic test
3. Auto-merge si minor/patch (gate manuel sur majors)
4. Auto-tag patch sur push-to-main (mathieudutour/github-tag-action)
5. Renovate côté `my-kluster` détecte le nouveau tag et propose un bump de `targetRevision: vX.Y.Z`
6. Merge côté `my-kluster` → ArgoCD sync (< 1 h)
7. CronJob run dans le créneau suivant (max 4 h pour arrconf/configarr)

## Operator runbook

### Snapshot avant un test risqué (toujours)

```bash
# 1. Lancer les port-forwards dans un terminal séparé (voir tools/snapshot/README.md)
# 2. Exporter les API keys dans l'env
# 3. Lancer le snapshot
./tools/snapshot/snapshot.sh
```
Output : `snapshots/baseline-YYYY-MM-DD/<app>/<resource>.json` versionné Git (lossless, anti-leak audit). Voir [ADR-6](./spec.md#adr-6).

### Forcer un run arrconf

```bash
# Smoke job manuel (utile en post-cutover et pour drift demos)
kubectl -n selfhost create job --from=cronjob/arrconf arrconf-manual-$(date +%s)
kubectl -n selfhost logs job/arrconf-manual-<timestamp> -f
```

### Diagnostiquer un drift

```bash
# Côté arrconf (reconciler)
arrconf diff --apps sonarr,radarr,prowlarr
# Côté snapshot raw (Bash + curl + jq)
./tools/snapshot/snapshot.sh --output snapshots/forensic-$(date +%FT%H%M)/
diff -r snapshots/baseline-<date>/ snapshots/forensic-<date>/
```

### Rollback du cutover (post-Phase 4)

Si le cutover crée des regressions inattendues : `git revert` la PR côté `my-kluster` qui ajoute `arr-stack-app.yaml` + supprime les 10 unit Apps. ArgoCD restore les 10 Applications individuelles. Les PVCs survivent (`Retain` policy + `existingClaim` discipline). Voir [`spec.md`](./spec.md) D-04-CUTOVER-04.
