## The Fix — `merge_fields_for_put` Diff

The GREEN edit adds a parallel privacy lookup (since `FieldKV.privacy` has `exclude=True`) and an omit-credential branch BEFORE the existing empty-value-substitute branch:

```python
def merge_fields_for_put[T: BaseModel](current: T, desired: T) -> dict[str, Any]:
    """... (existing docstring + new paragraph documenting D-02.2-AUTH-REGRESSION + ordering invariant) ..."""
    cur_dump = current.model_dump(exclude_none=True)
    des_dump = desired.model_dump(exclude_none=True, exclude=_READ_ONLY_FIELDS)
    cur_by_name = {f["name"]: f for f in cur_dump.get("fields", [])}
    # NEW: parallel privacy lookup from pydantic model (FieldKV.privacy is exclude=True
    # so it does NOT appear in cur_dump — read it directly off the model instances).
    cur_privacy_by_name: dict[str, str | None] = {
        f.name: f.privacy for f in getattr(current, "fields", [])
    }
    merged_fields: list[dict[str, Any]] = []
    for des_f in des_dump.get("fields", []):
        cur_f = cur_by_name.get(des_f["name"])
        cur_privacy = cur_privacy_by_name.get(des_f["name"])
        # NEW v0.1.5 / D-02.2-AUTH-REGRESSION omit-credential branch (FIRST in loop):
        if cur_privacy in ("password", "userName"):
            log.info(
                "merge_field_omitted_credential",
                name=des_f["name"],
                privacy=cur_privacy,
            )
            continue
        # EXISTING Phase 2.1 empty-value preserve-cluster branch (UNCHANGED):
        v = des_f.get("value")
        if v == "" or v is None:
            if cur_f is not None and cur_f.get("value") not in ("", None):
                merged = dict(des_f)
                merged["value"] = cur_f["value"]
                log.info("merge_field_preserved", name=des_f["name"])
                merged_fields.append(merged)
                continue
        merged_fields.append(des_f)
    des_dump["fields"] = merged_fields
    return des_dump
```

Net change: **+41 / -1 in differ.py** (extended docstring + new branch + parallel lookup). The Phase 2.1 contract for non-credential fields is untouched.

### Key implementation note (deviation-grade documentation)

The plan's AFTER-shape sketch in `<interfaces>` used `cur_f.get("privacy")` against the `cur_dump`-derived dict, but `FieldKV.privacy` is declared with `exclude=True` in `arrconf/resources/sonarr/download_client.py` — that's the long-standing convention to keep UI metadata out of PUT bodies. Consequently `current.model_dump()` strips `privacy`, and the dict-based check would always read `None`, leaving the regression unfixed.

Discovered during the first GREEN-iteration test run (both RED tests still failed — output showed `merge_field_preserved` events for `username` and `password`, which means the omit branch never matched). Fixed by reading `f.privacy` directly off the `FieldKV` pydantic instances (`for f in current.fields`). This keeps the FieldKV exclude policy unchanged — minimum surface area, no resource-model edits needed. Documented inline in the differ.py docstring + commit message + the SUMMARY decisions list above.

This is a Rule 1 auto-fix (the plan's interface sketch was incorrect for the underlying pydantic model contract) — not a deviation from intent, just a correction to make the implementation match the planned strategy.

## Audit Event Contract (T-02.2-08-01)

Every omitted credential entry emits exactly one structlog event:

```jsonc
{"name": "username", "privacy": "userName", "event": "merge_field_omitted_credential", "level": "info", "timestamp": "..."}
{"name": "password", "privacy": "password", "event": "merge_field_omitted_credential", "level": "info", "timestamp": "..."}
```

Payload is **metadata-only** — no `value` field, no API mask token (`********`), no in-tree redaction sentinel (`***REDACTED***`), no real credential like the cluster's literal `"admin"` username. Verified by:
