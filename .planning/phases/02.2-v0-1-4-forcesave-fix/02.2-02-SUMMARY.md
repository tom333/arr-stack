# Phase 02.2 Plan 02: TDD forceSave injection Summary

Two atomic TDD commits closing the architectural finding D-02.1-06 at the HTTP client layer: `_ArrV3Client(ArrApiClient)` mixin overrides `put()` to inject `?forceSave=true` and emit `put_force_save_used` structlog event; `SonarrClient` re-parented onto it. Phase 3 RadarrClient/ProwlarrClient inherit the behavior for free (zero-discipline default per D-02.2-02).

### Architectural option chosen — Option (b) `_ArrV3Client` mixin

Rationale (one-line per the plan's pre-locked rationale): the explicit type hierarchy is self-documenting and matches the *arr v3 / qBit / Jellyfin natural fault lines — Phase 5/7 contributors reading `client_base.py` cold see `_ArrV3Client` is *arr v3 territory and inherit from `ArrApiClient` directly for non-*arr-v3 semantics. ### Code shape (after GREEN)

```python
class _ArrV3Client(ArrApiClient):
    """Common base for *arr v3 REST clients (Sonarr/Radarr/Prowlarr).

    Always sends ``forceSave=true`` on UPDATE PUTs (D-02.2-01 / ADR-8).
    """

    def put(self, path: str, id: int, json: Any, **kwargs: Any) -> Any:
        params = dict(kwargs.pop("params", None) or {})
        params.setdefault("forceSave", "true")  # caller-supplied wins on conflicts
        kwargs["params"] = params
        log.info("put_force_save_used", path=path, id=id)
        return self._request("PUT", f"{path}/{id}", json=json, **kwargs).json()


class SonarrClient(_ArrV3Client):  # parent changed from ArrApiClient
    api_path = "/api/v3"
    name = "sonarr"
```

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 — Bug] Existing UPDATE-route regexes anchored on `\d+$` stopped matching when URLs gained `?forceSave=true` query string**

- **Found during:** Task 2.2 GREEN — first full pytest run after implementation
- **Issue:** Two pre-existing tests (`test_update_existing_download_client`, `test_update_preserves_redacted_credentials_in_put_body`) and my new RED test all used `url__regex=r"^http://sonarr\.test/api/v3/downloadclient/\d+$"`. respx matches against the full URL (including query string), and the `$` anchor immediately after `\d+` rejects any URL with `?forceSave=true` — resulting in `respx.models.AllMockedAssertionError: RESPX: <Request('PUT', '...?forceSave=true')> not mocked!` for all three UPDATE tests.
- **Fix:** Relaxed the regex to `\d+(?:\?.*)?$` (allows optional query string suffix) on all three tests. DELETE-route regexes (`\d+$`) are unchanged because DELETE doesn't carry forceSave (verified by Test 3).
- **Files modified:** `tools/arrconf/tests/test_reconcilers_sonarr.py` (3 regex sites updated)
- **Commit:** Folded into GREEN commit `4a24c5f` (the test-regex fix is required for the implementation to ship green; cannot land independently)
- **Pattern established:** PATTERNS.md "PUT URL regex pitfall" already warned against `url__regex` end-of-string anchors silently slipping past changes — Phase 02.2 is the first phase where that pitfall would have actually broken the build. Documented for Phase 3.

**2. [Rule 3 — Blocking] ruff N802 lint blocked the locked test names with `forceSave` camelCase**

- **Found during:** Task 2.2 GREEN — `uv run ruff check arrconf tests` after first GREEN attempt
- **Issue:** ruff's pep8-naming `N802` rule rejects function names with non-lowercase characters. The three new test names (`test_update_passes_forceSave_query_param`, `test_add_does_not_pass_forceSave_query_param`, `test_delete_does_not_pass_forceSave_query_param`) contain `forceSave` camelCase — locked by the plan acceptance criteria as a literal substring match.
- **Fix:** Added `# noqa: N802 — \`forceSave\` matches Sonarr API param literal (locked by D-02.2 plan)` on each of the three function definitions. The camelCase is intentional and load-bearing — it matches the Sonarr URL parameter `?forceSave=true` (the test asserts on this exact name as a string key); paraphrasing to snake_case would break the visual coupling between the test name and the assertion target.
- **Files modified:** `tools/arrconf/tests/test_reconcilers_sonarr.py` (3 noqa annotations added)
- **Commit:** Folded into GREEN commit `4a24c5f`
