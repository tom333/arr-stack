<objective>
TDD code change: introduce a `_ArrV3Client(ArrApiClient)` intermediate class that overrides `put()` to inject `?forceSave=true` on every UPDATE PUT, emit a `put_force_save_used` structlog event, and have `SonarrClient` inherit from it. The fix sits at the HTTP client layer (D-02.2-02) so Phase 3's `RadarrClient` and `ProwlarrClient` inherit it for free with zero per-reconciler discipline burden.

**Architectural option chosen: Option (b) — `_ArrV3Client` mixin** (per planner's call documented here).

Rationale: PATTERNS.md surfaces three options. (a) class flag is smallest diff but inverts the default scope (qBit/Jellyfin must remember to set `False`). (b) intermediate class is most explicit about scope at the type level — a contributor at Phase 5 (qBittorrent) reading `client_base.py` cold sees `_ArrV3Client` is *arr v3 territory and inherits from `ArrApiClient` directly instead. The explicit type hierarchy is self-documenting and matches the *arr v3 / qBit / Jellyfin natural fault lines. (c) is rejected by CONTEXT.md "Discretion" (forgotten-kwarg failure mode at Phase 3 boundary).

Closes the architectural finding D-02.1-06 from Phase 2.1: Sonarr's `merge_fields_for_put` faithfully preserves the API mask `********` for password fields; without `forceSave=true`, Sonarr's pre-save validation re-authenticates against `********` literally and returns HTTP 400 on any real-change PUT. The forceSave bypass tells Sonarr: "we're a trusted controller, skip the UI-grade validation." Trade-offs are documented in ADR-8 (Plan 03).

Purpose: Unblock Phase 3 (Radarr/Prowlarr automated drift correction) by making PUT-with-mask succeed in the normal reconcile path. Closes the correction half of REQ-drift-detection cleanly (Phase 2.1 closed it via operator manual nudge — Phase 02.2 closes it via fully-automated reconcile, verified in Plan 06).

Output: Two atomic TDD commits (RED test + GREEN implementation), all existing tests still pass, no behavior change for ADD/DELETE.

# NEW class to introduce (between ArrApiClient and SonarrClient):
class _ArrV3Client(ArrApiClient):
    """Common base for *arr v3 REST clients (Sonarr/Radarr/Prowlarr).

    Always sends ``forceSave=true`` on UPDATE PUTs (D-02.2-01 / ADR-8).
    arrconf is the trusted controller — Sonarr's UI-grade pre-save validation
    (which re-authenticates against masked passwords) is a tax with no signal
    value in our context.
    """

    def put(self, path: str, id: int, json: Any, **kwargs: Any) -> Any:
        """HTTP PUT /{path}/{id} with ``forceSave=true`` always set."""
        params = dict(kwargs.pop("params", None) or {})
        params.setdefault("forceSave", "true")
        kwargs["params"] = params
        log.info("put_force_save_used", path=path, id=id)
        return self._request("PUT", f"{path}/{id}", json=json, **kwargs).json()
