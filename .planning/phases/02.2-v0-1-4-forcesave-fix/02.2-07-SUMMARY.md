## Decisions Made

1. **Hoisted stdlib + pytest imports to top of test file** — chose the cleaner shape from the plan's two alternatives. The 15 pre-existing test FUNCTION BODIES remain byte-identical (verified 0 deletions in diff); only the imports block was extended additively. Rationale: ruff E402 is a hard CI gate per CLAUDE.md.
2. **Anti-leak coverage extended to JSON object keys (auto-fix Rule 2)** — `with_entries(.key |= gsub(...))` applied to `qbittorrent/sync_maindata.json[trackers]` because `walk + gsub` only rewrites string values; the passkey hex was leaking as object keys in the trackers map.
3. **Username field treated uniformly with password** — even though the cluster username `"moi"` (and the in-tree fixture's `"admin"`) are real values rather than mask tokens, the omit-by-metadata strategy treats `privacy=password` and `privacy=userName` identically. The "absence-as-protection" semantic is the contract — Sonarr preserves stored values when fields are absent from the PUT body, regardless of whether the cluster value happens to also be a mask. This is documented in the in-tree-redacted-mask test's docstring.

### Auto-fixed Issues

**1. [Rule 2 - Missing Critical] Anti-leak coverage gap: `sync_maindata.json[trackers]` object keys**
- **Found during:** Task 7.1 (anti-leak audit step 6)
- **Issue:** Initial redaction sequence (Plan steps 5a/5b/5c) only covered `torrents_info.json` and `app_preferences.json`. The `sync_maindata.json` endpoint also returns tracker URLs — but as JSON object KEYS in the `.trackers` map (e.g., `"https://c411.org/announce/feed603a..."`). The plan's `walk + gsub` recipe only rewrites string VALUES, not object keys. Anti-leak audit caught the leak: `qBit passkey raw matches: 1` on first run.
- **Fix:** Added a 4th redaction pass: `jq '.trackers |= (with_entries(.key |= (gsub("/announce/[a-fA-F0-9]{16,}"; "/announce/PLACEHOLDER_PASSKEY"))))' sync_maindata.json` — rewrites each tracker URL key in the `.trackers` map.
