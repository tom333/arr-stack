### D-08 Mitigation Note

After refreshing the catalog (re-running `fetch-trash-metadata.sh`), run
`arrconf apply --dry-run` against the current cluster config to verify configarr
accepts the new IDs. If configarr logs `Custom format not found`, its TRaSH cache
may be stale — the next CronJob run will re-clone and resolve. No CI cross-check
is needed at homelab scale.

### configarr Version Requirement

Baked QP catalog uses Feb-2026+ TRaSH item ordering (highest quality first).
Requires **configarr >= v1.22.0** (released 2026-02-20). If configarr is ever
downgraded below v1.22.0, re-bake the QP catalog from a pre-2026-02-20 TRaSH commit.

### Scope

- Custom format IDs with `fr-*` prefix (e.g. `fr-vff`, `fr-vfi`, `fr-vfq`) are
  **local-only** definitions in `configarr.yml#customFormatDefinitions[]`. They are
  NOT present in this catalog and are rendered with a "custom" badge in the UI.
- No quality_definition or media_naming catalogs are baked (out of scope — managed
  by configarr natively via TRaSH-Guides at apply time).
